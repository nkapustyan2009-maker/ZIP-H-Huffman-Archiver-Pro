# ZIP-H Huffman Archiver Pro

## Опис проєкту
Графічний застосунок, розроблений мовою C++ з використанням Windows API, призначений для стиснення та розпакування файлів будь-якого формату за допомогою математичного алгоритму кодування Хаффмана. Програма забезпечує високу точність обробки бінарних даних, динамічне логування процесів у реальному часі та оптимізована для операційної системи Windows.

---

## Мета проєкту
Створення надійного, швидкодіючого та інтуїтивно зрозумілого інструменту для:
- зменшення обсягу файлів на диску без втрати інформації
- вивчення практичного застосування жадібних алгоритмів та структур даних
- освоєння низькорівневої роботи з бінарними файловими потоками та побітовим введенням-виведенням
- побудови кастомних графічних інтерфейсів за допомогою чистих системних компонентів Windows

---

## Технологічний стек
- Мова програмування: C++ (стандарт C++11 / C++14)
- Графічний інтерфейс: WinAPI (Windows SDK), Rich Edit Control
- Динамічні бібліотеки: Msftedit.dll
- Алгоритми та структури даних: Алгоритм Хаффмана, черга з пріоритетами (std::priority_queue), бінарні дерева, префіксне кодування, побайтовий частотний аналіз
- Інструменти розробки: Visual Studio / VS Code, Git, GitHub

---

## Основні можливості
- Вибір вхідного об'єкта обробки будь-якого розширення через системний діалог GetOpenFileNameW
- Побайтовий аналіз частот появи символів у файлі та побудова оптимального дерева Хаффмана
- Ефективна компресія даних та збереження результату в архівний файл із фірмовою сигнатурою HUFF3 та розширенням .huff
- Повна коректна декомпресія архіву з урахуванням службових метаданих заголовка та бітів вирівнювання (padding bits)
- Відновлення оригінального об'єкта без побітових втрат із додаванням суфікса _extracted
- Інтерактивне покрокове виведення етапів виконання операцій у кастомне текстове поле логування RichEdit
- Функція миттєвого очищення робочої області логу повідомленням WM_SETTEXT
- Стійкість системи до граничних навантажень: захист від обробки порожніх файлів (0 байт) та обмеження мінімального розміру вікна через WM_GETMINMAXINFO

## Особливості
Проєкт реалізовано з урахуванням високих вимог до продуктивності та інтерфейсу:
- повністю кастомний графічний рендеринг елементів керування, округлих кнопок та фонів (Owner-Drawn Controls)
- інтегрована темно-синя колірна схема (Dark Theme) для зниження втоми очей користувача
- рекурсивне автоматичне звільнення динамічної пам'яті від вузлів дерева при закритті програми
- модульна та чиста архітектура вихідного коду, готова до подальшої кросплатформної міграції ядра
